mammut::task::ThreadHandler * mammut::utils::Thread::getThreadHandler ( ) const

Returns the thread handler associated to this thread. It is automatically released when this thread is joined.

Returns
The thread handler associated to this thread. If this thread is not yet started (or if it finished its execution) NULL is returned.
void mammut::utils::Thread::join ( )

Joins this thread. After this operation, the handler obtained with getThreadHandler() call is no more valid.

virtual void mammut::utils::Thread::run ( )
pure virtual

This is the function that will be executed by the thread. To create a custom thread, is sufficient to extend this class and implement this pure member function.

Implemented in mammut::topology::SpinnerThread, and mammut::energy::CounterCpusLinuxRefresher.

bool mammut::utils::Thread::running ( )

Checks if the thread is still running.

Returns
True if the thread is still running, false otherwise.
void mammut::utils::Thread::start ( )

Starts this thread. ATTENTION: When the object is deallocated the thread will keep running. It must EXPLICITLY be joined before the object goes out of scope.
